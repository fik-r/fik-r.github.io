Alasan murtad ke kotlin dari java

2020-04-04
  • Kotlin1

Awal sebelum gua kenal kotlin, bahasa pemrograman yang biasa gua pakai buat develop android ya Java. Cuman lama kelamaan gua ngerasa penulisan kode di Java itu terlalu bertele-tele atau kepanjangan dan sering kali kena nullpointer exception sehingga aplikasi sering crash. Hingga akhirnya gua kenal dan memutuskan untuk murtad ke Kotlin. Nguli Ngoding jadi jauh lebih fun, cepet dan productive.

Lebih ringkas

Yang artinya hanya butuh sedikit line code untuk solve suatu problem ketimbang menggunakan java. Tentu saja ini meningkatkan kualitas kode kita jadi lebih maintainability dan readability, yang artinya programmer bisa menulis, membaca, atau merubah code lebih efisien dan efektif.

Bisa menggunakan semua Java framework dan library yang ada.

Ga usah khawatir soal framwork atau library lagi karna semua java framework dan library compatible dengan kotlin.

Mudah dipelajari

Buat kalian yang udah sering gonta2 bahasa pemrograman atau pun baru migrasi dari java. Belajar kotlin udah pasti gampang banget. Karena Kotlin didesain untuk memakai elemen-elemen yang sudah ada dengan perbaikan dibagian tertentu sehingga proses ngoding menjadi lebih fun.

Null Safety

Kalian ga usah khawatir lagi soal NullPointerException di kotlin. Karena compiler akan mendeteksi adanya kemungkinan error sebelum membuat APK atau di run.

Biar kalian makin mantap untuk murtad ke kotlin. coba liat video dibawah
